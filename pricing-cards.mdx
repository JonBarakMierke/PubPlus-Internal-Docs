Reusable pricing section with two distinct layouts. Renders as `<x-sections.pricing>`.

## File Structure

```text
resources/views/components/sections/pricing/
├── index.blade.php                       # Router — resolves variant, theme, and shared props
├── variants/
│   ├── cards.blade.php                   # Independent plan cards in a responsive grid
│   └── comparison.blade.php              # Single comparison table, one row per shared feature
└── partials/
    ├── header.blade.php                  # Shared eyebrow/heading/subtext/actions header (both variants)
    ├── plan.blade.php                    # One plan card — `cards` variant only
    └── comparison-plan.blade.php         # One plan's header cell — `comparison` variant only
```

---

## Props

| Prop | Type / Values | Default | Notes |
| :-- | :-- | :-- | :-- |
| `variant` | `cards` \| `comparison` | `cards` | Picks which file in `variants/` renders. |
| `container` | any Tailwind `max-w-*` class | `max-w-6xl` | Controls the max width of the section content. |
| `theme` | `page` \| `light` \| `dark` \| `navy` | `page` | Controls the **non-featured** surface, text, and border colors as a matched set. `page` sits on the plain page background and auto-adapts to light/dark; the others are for placing the section on a colored band. See [Themes](#themes). |
| `columns` | `2` \| `3` \| `4` | `2` | `cards `**variant only.** Plans per row on desktop, before any plan opts out with `span: full`. |
| `plans` | array of plan arrays | `[]` | Shape differs per variant — see [Plan Shape — `cards` Variant](#plan-shape-cards-variant) and [Plan Shape — `comparison` Variant](#plan-shape-comparison-variant). |
| `features` | array of strings | `[]` | `comparison `**variant only.** Shared row labels shown down the left column. |

A `featured` plan always gets a fixed elevated treatment regardless of `theme` — that's the point of marking a plan featured, not something a theme should be able to turn off.

---

## Slots

| Slot | Required | Description |
| :-- | :-- | :-- |
| `eyebrow` | No | Small uppercase label/pill above the heading. |
| `heading` | No | Main `<h2>` (rendered via `x-ui.heading`). |
| `subtext` | No | Supporting paragraph under the heading. |
| `actions` | No | Button(s) shown below the subtext. |

All slots are optional and independently `@isset`-guarded. Unlike the CTA section, this header is **always centered** — there's no `start`-aligned layout option, and the heading slot doesn't currently forward a user-supplied `class` through to `x-ui.heading`.

## Themes

Themes are a fixed set defined once in `index.blade.php`, shared by both variants. There is **no free-form `bg` prop** — this keeps every pricing section on-brand.

| Theme | Background / Border | Text | Use case |
| :-- | :-- | :-- | :-- |
| `page` | `bg-white` / `border-neutral-200` (auto dark: `bg-neutral-900` / `border-neutral-800`) | Neutral 900 / Neutral 50 dark | Default — section sits directly on the plain page surface. |
| `light` | `bg-white` / `border-neutral-200` | Neutral 900 | Section placed on a light colored band. |
| `dark` | `bg-white/5` / `border-white/10` | White / Neutral 300–400 | Section placed on a dark colored band. |
| `navy` | `bg-white/5` / `border-white/10` | White / Neutral 300 |

**To add a new theme:** add one entry to the `$themes` array in `index.blade.php` with `card`, `name`, `desc`, `price`, `suffix`, `divide`, `text`, `muted`, and `zebra` keys. It becomes available to both variants immediately — no changes needed in the variant files.

## Plan Shape — `cards` Variant

Each entry in `:plans` is rendered as an independent card by `partials/plan.blade.php`.

| Key | Type | Notes |
| :-- | :-- | :-- |
| `name` | string | Card title. |
| `description` | string | Small caption under the title. |
| `price` | string | Large display price, e.g. `'$25'`. |
| `priceSuffix` | string | Optional, sits next to the price, e.g. `'per month'`. |
| `features` | array | List of `['text' => ..., 'included' => bool]`. `included` defaults to `true`. A plain string (`'Trackable QR code'`) works the same as `['text' => 'Trackable QR code']`. |
| `cta` | array | `['label' => ..., 'url' => ...]`. Optional — omit to render a card with no button. |
| `featured` | bool | Default `false`. Renders a solid accent-filled card (`bg-grad-accent-left`) instead of the themed surface. |
| `span` | `'full'` | Optional. Breaks the card out of the column grid to span the full row width, and switches its own feature list to two columns (`sm:grid-cols-2`). |

### Full-Width Callouts (`span: full`)

Any plan with `'span' => 'full'` ignores the `columns` prop entirely and spans every column via `md:col-span-full`. This is how a callout tier (e.g. a "Growth" or "White Labeling" plan) breaks out from a row of regular side-by-side cards — combine it with `'featured' => true` for the full look.

---

## Plan Shape — `comparison` Variant

Each entry in `:plans` becomes one header cell (name/price/CTA), rendered by `partials/comparison-plan.blade.php`. Which feature rows show a check mark for that plan is driven by `included`, matched against the top-level `:features` array.

| Key | Type | Notes |
| :-- | :-- | :-- |
| `name` | string | Plan name shown above the price. |
| `price` | string | Large display price, e.g. `'$45'`. |
| `priceYearly` | string | Optional. If **any** plan in `:plans` sets this, the yearly toggle appears — see [Yearly Toggle](#yearly-toggle-comparison-variant). Plans without their own `priceYearly` keep showing `price` in both toggle states. |
| `priceSuffix` | string | Optional, defaults to `'Per Month'`. |
| `cta` | array | `['label' => ..., 'url' => ...]`. Optional. |
| `featured` | bool | Default `false`. Pops the header cell above/below the header row as an elevated `bg-neutral-900` card — see [Featured Column](#featured-column-comparison-variant). |
| `included` | array of strings | Feature labels this plan includes. Each entry **must match a string in the top-level `:features` array exactly** — there's no fuzzy or case-insensitive matching. Anything in `:features` but not in a plan's `included` renders as a blank cell for that plan. |

### Featured Column (`comparison` variant)

The `featured` plan's header cell rises above the header row (`-mt-6`/`-mt-8`) and dips slightly below it (`-mb-6`) as a dark, elevated card — the rest of the table (every feature row, for every plan) stays visually plain regardless of which column is featured.

This only works because the header row and the feature rows are **two separate CSS grids** stacked with an `mt-8` gap between them, not one continuous grid — that gap is exactly where the featured cell's `-mb-6` overlap lands, so it can never cover a check mark. If you're adjusting these values, keep the header-to-toggle gap (`mb-14`/`mb-16` on the toggle wrapper in `variants/comparison.blade.php`) larger than the featured cell's `-mt-*` pull-up, or the dark card will paint over the toggle instead (it comes later in DOM order, so it wins the overlap).

### Yearly Toggle (`comparison` variant)

A plain two-button Alpine pill (`x-data="{ yearly: false }"` on the section root), not `x-ui.switch` — that component derives its form `name` from an `x-model` attribute and re-spreads `$attributes` (including that same `x-model`) onto an inner hidden input, which ended up fighting the outer `x-model` for control of `$yearly`. Every price with a `priceYearly` value swaps via `x-show="!yearly"` / `x-show="yearly"`; the toggle itself only renders at all if at least one plan defines `priceYearly`.

## Extending This Component

1. Create `resources/views/components/sections/pricing/variants/{name}.blade.php`.
2. Add an entry to the `$variants` array in `index.blade.php`:
   ```php
   $variants = [
       'cards'      => 'components.sections.pricing.variants.cards',
       'comparison' => 'components.sections.pricing.variants.comparison',
       'new-name'   => 'components.sections.pricing.variants.new-name',
   ];
   ```
3. The new file automatically receives every prop and shared computed value (`$theme`, `$palette`, `$eyebrowTheme`, `$gridCols`, `$container`, `$plans`, `$features`, `$eyebrow`, `$heading`, `$subtext`, `$actions`, etc.) via the shared `@include($view)` — no extra wiring needed.
4. Reference `$palette['card']`, `$palette['name']`, `$palette['text']`, etc. for surface/text colors so the new variant respects themes automatically, rather than hardcoding colors.

To add a new theme instead of a new variant, see [Themes](#themes) above — it only requires one new entry in the `$themes` array, not a code change to either variant file.
