Reusable, condensed, guest-facing lead-capture form. Collects a name and email, emails the production team, and captures a PostHog analytics event. Every page-specific detail ‚Äî copy, analytics source, email subject, styling theme ‚Äî is passed in when the component is placed, so one component \+ one mailer serves every landing page. Rendered as `<livewire:components.public.lead-form>`.

## Props

All props are passed as attributes on the `<livewire:...>` tag (kebab-case). `source` and `sourceLabel` are required; everything else has a sensible default.

| Prop | Type | Default | Notes |
| :-- | :-- | :-- | :-- |
| `source` | `string` | ‚Äî **required** | Analytics stem. Emits a PostHog event named `{source}_lead_submitted` (e.g. `paperlink` ‚Üí `paperlink_lead_submitted`). |
| `source-label` | `string` | ‚Äî **required** | Human-readable origin used in the notification email, e.g. `the PaperLink page`. |
| `subject` | `string` | `New Lead | PubPlus Site` | Subject line of the notification email. |
| `theme` | `dark` \| `light` | `dark` | Input styling. See [Themes](#themes). |
| `button-label` | `string` | `Get Started` | Text on the submit button. |
| `success-title` | `string` | `Request received!` | Heading shown after a successful submit. |
| `success-text` | `string` | `Thanks ‚Äî we'll reach out shortly.` | Body copy shown after a successful submit. |
| `helper-text` | `string` | `No spam ‚Äî just a quick note to follow up.` | Fine-print line under the inputs. |

All configuration props are `#[Locked]` ‚Äî they're set on `mount()` and can't be tampered with from the client, but persist across Livewire requests (`submit()` runs on a later request).

## Themes

The `theme` prop only controls how the **inputs** look, so the form sits correctly on either a dark or light band. Everything else (button, success check, copy) is identical across themes.

| Theme | Inputs | Use case |
| :-- | :-- | :-- |
| `dark` (default) | Glassy ‚Äî translucent white fill, blurred backdrop, light text | Place on a genuinely dark section/band. |
| `light` | Solid white fill, dark text | Place on a light or gradient band. |

Any value other than `light` is treated as `dark`.

## Behavior

- **Validation** ‚Äî `name` (required, 2‚Äì100 chars) and `email` (required, valid email, ‚â§255 chars). Inline, per-field error messages render under each input; the field border turns red on error.
- **Submit** ‚Äî On a valid submit the component:
  1. Emails `config('mail.internal_quotes_to')` (falls back to `jon@pubplus.net`) via the queued `LeadNotification` mailable, using the passed `subject` and `source-label`.
  2. Captures a PostHog event `{source}_lead_submitted` keyed on the submitter's email.
  3. Resets the fields and flips to the success state.
- **Loading state** ‚Äî The button shows a spinner and ‚ÄúSending‚Ä¶‚Äù, and disables itself while the request is in flight.
- **Success state** ‚Äî Replaces the form with a check badge plus `success-title` / `success-text`.
- **Multiple per page** ‚Äî Field `id`s are namespaced with the Livewire component id, so several lead forms can safely coexist on one page.

## The Notification Email

Submissions send `App\Mail\Public\LeadNotification` ‚Äî a queued (`ShouldQueue`) markdown mailable rendered from `mail/public/lead/lead-mail.blade.php`.

- **To:** `config('mail.internal_quotes_to')` ‚Üí falls back to `jon@pubplus.net`.
- **Subject:** the `subject` prop.
- **Body:** the submitted `name` / `email`, plus the `source-label` so the team knows which page the lead came from.

To change where leads are delivered, set `mail.internal_quotes_to` ‚Äî no component change needed.

## Adding This Form to a New Page

1. Drop `<livewire:components.public.lead-form>` wherever the CTA lives.
2. Set `source` (a unique analytics stem) and `source-label` (a human origin string) ‚Äî these are required.
3. Pick a `theme` that matches the band it sits on (`dark` or `light`).
4. Override any copy props (`subject`, `button-label`, `success-*`, `helper-text`) to fit the page.

No new component, mailer, or route is needed ‚Äî the same component and mailable serve every landing page.
